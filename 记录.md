## 技术决策

### 1. 输出格式
- 主要输出：**YAML rule-provider** 文件（Clash Premium + Clash.Meta/mihomo 兼容）
- GeoIP：**MMDB + ASN**（不使用过时的 .dat 格式）
- MMDB/ASN 来源：`Loyalsoldier/geoip` GitHub Release（免注册直接下载），混合去重

### 2. 构建语言：Go
- 用户不喜欢 Python（竞赛后遗症），C++ 过重
- Go 优势：Clash 生态一致、`net` 标准库原生支持 IP/CIDR、编译为单二进制 CI 直接跑、可读性好

## 核心架构设计

### 数据流
```
sources.yaml 配置
    ↓
并发 Fetch 所有上游 URL（3 次重试）
    ↓
按 parser 类型解析为 []Entry{Type, Payload}
    ↓
按输出规则聚合多个源 → FilterByBehavior → Merge 去重排序
    ↓
按 behavior 格式化写入 YAML（domain/ipcidr/classical）
```

### Entry 类型定义（internal/types/types.go）
```go
type Entry struct {
    Type    string // DOMAIN, DOMAIN-SUFFIX, DOMAIN-KEYWORD, IP-CIDR, IP-CIDR6
    Payload string
}
```

### 解析器类型
| 解析器名 | 输入格式 | 输出 Entry.Type |
|---------|---------|----------------|
| `cidr` | 每行一个 CIDR | IP-CIDR / IP-CIDR6 |
| `dnsmasq` | `server=/domain/ip` | DOMAIN-SUFFIX |
| `domain-list` | 每行一个域名 | DOMAIN-SUFFIX |
| `gfwlist` | Base64 编码的 ABP 规则 | DOMAIN-SUFFIX |
| `v2fly-dlc` | `domain:/full:/keyword:` 格式 | DOMAIN-SUFFIX / DOMAIN / DOMAIN-KEYWORD |
| `classical` | `TYPE,payload` 格式 | 原样保留 |
| `static` | 配置内嵌列表 | 自动判断 |

### Writer 格式化规则
- `behavior: domain` → DOMAIN-SUFFIX 输出 `.domain`，DOMAIN 输出 `domain`，DOMAIN-KEYWORD 跳过
- `behavior: ipcidr` → 直接输出 CIDR 字符串
- `behavior: classical` → 输出 `TYPE,payload`

## sources.yaml 配置结构

```yaml
upstream:
  源名:
    url: "https://..."
    parser: "解析器类型"
    entries: [...]  # 仅 static 类型使用

output:
  规则名:
    filename: "输出文件名.yaml"
    description: "描述"
    behavior: "domain|ipcidr|classical"
    sources: ["源名1", "源名2", ...]

geoip:
  files:
    - url: "https://github.com/Loyalsoldier/geoip/releases/latest/download/Country.mmdb"
      filename: "Country.mmdb"
    - url: "https://github.com/Loyalsoldier/geoip/releases/latest/download/Country-asn.mmdb"
      filename: "Country-asn.mmdb"
```
